# Critic Agent

**Role:** UX Critic / Heuristic Evaluator
**Model:** `gemini-2.5-flash`
**Port:** `8004`
**Protocol:** A2A

## Overview
The Critic Agent evaluates the mockups generated by the UX Designer Agent. It verifies that the designs align with the Architectural Decision Records (ADRs) and assesses them across four dimensions: Alignment, Usability, Accessibility, and Aesthetics. It issues an APPROVED or REJECTED verdict that controls the design iteration loop.

## Workflow
1. Reads the ADRs (`adr_collection.md`) to establish the architectural constraints.
2. Reads the mockup descriptions (`ux_mockup.md`) and optionally inspects the visual implementations in Stitch.
3. Evaluates the flow and design against the four dimensions.
4. Assigns a Pass/Fail score to each dimension.
5. Generates a detailed evaluation report.
6. Submits the final review status to either approve the design or trigger another iteration.

## Input
- `adr_collection.md`: Technical constraints (from Architect).
- `ux_mockup.md`: Screen specifications and designs (from UX Designer).

## Output
- `ux_feedback.md`: A markdown evaluation report detailing scores, findings, and actionable feedback.

## Tools
- `get_document`: Retrieves upstream artifacts.
- `save_document`: Saves the `ux_feedback.md` artifact.
- `submit_review`: Submits the final decision (`APPROVED` or `REJECTED`) back to the Orchestrator.
- `stitch_toolset`: MCP toolset to inspect Google Stitch projects and screens.

## Requirements
Requires a valid `STITCH_API_KEY` environment variable.

## Running Locally
```bash
export STITCH_API_KEY="your-api-key"
uv sync
uv run uvicorn app:app --host 0.0.0.0 --port 8004
```
