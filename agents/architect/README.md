# Architect Agent

**Role:** System Architect
**Model:** `gemini-2.5-flash`
**Port:** `8002`
**Protocol:** A2A

## Overview
The Architect Agent is the second step in the Agentic Studio pipeline. It specializes in system design, technology evaluation, and architectural decision-making. It produces Architecture Decision Records (ADRs) that capture the rationale behind key technical choices based on the User Stories generated by the Analyst.

## Workflow
1. Reads all User Stories to identify cross-cutting technical concerns.
2. Groups related concerns into architectural decisions (e.g., "Frontend Framework", "Auth Strategy").
3. Evaluates alternatives with explicit pros and cons for each decision.
4. Selects the option that best serves the user stories' requirements.
5. Documents the consequences, positive trade-offs, and accepted risks.

## Input
- `user_stories.md`: The User Stories artifact generated by the Analyst Agent.

## Output
- `adr_collection.md`: A structured markdown document containing the Architecture Decision Records (ADRs). This defines the technical constraints that the UX Designer must respect.

## Tools
- `get_document`: Retrieves the `user_stories.md` artifact.
- `save_document`: Saves the generated `adr_collection.md` artifact.

## Running Locally

uv sync
